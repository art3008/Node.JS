WebPack - это сборщик модулей. Он анализирует модулю приложения, создает граф зависимостей, 
затем собирает модули в правильном порядке в один или более бандл, на который может ссылаться файл "index.html"

Бандл - это файл JavaScript, содержащий код из всех модулей проекта и объединенных в правильном порядке.
Когда вебпак создает граф зависимостей, он не выполняет исходный код, а объединяет модули и их зависимости в бандл

Для установки WebPack в терминале прописываем команду:

== npm install webpack webpack-cli ==

С помощью веб - пака можно разделить код на production и development

mode: 'development'/'production' - указывает веб-пакету, что необходимо соответствующим образом использовать встроенные оптимизации

context: path.resolve('./', 'webpack', 'src') - входная точка. с какой директорией webpack будет работать

plugins: - определённые дополнения к webpack'у, делают то, что не умеет webpack (плагины должны быть классами)

module: - модули
        rules: - правила для модулей
            loader'ы работают с файлами, с которыми не может обрабатывать webpack

devServer: { - запускает сервер на том порте, который мы указали
        port: 4200,
        open: true
    }
